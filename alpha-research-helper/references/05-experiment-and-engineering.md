---

## 1. 三阶段推进法

```
阶段一：过拟合验证
  → 小数据集上验证模型能 loss → 0
  → 确认代码无 bug、数据 pipeline 正确

阶段二：泛化调优
  → 控制变量法逐个优化超参
  → 目标：test RankIC 稳步提升

阶段三：RankIC 冲刺
  → 精调、集成、多 seed 验证
  → 准备上线或提交
```

---

## 2. 控制变量法 + Round 制

每轮搜索一个或一组相关参数：

```
Round 1: 模型档位 × 损失函数
Round 2: 正则化 (dropout × aug_noise)
Round 3: 学习率 × 批量大小
Round 4: 训练策略 (warmup, patience, seed)
...
```

**规则**：
- 只变实验参数，固定其他
- 多 seed 取均值 ± 标准差
- 改进必须显著大于 seed 方差才采纳
- 最优点在边界 → 扩大范围重搜

---

## 3. 何时 kill 实验

观察实时 IC 走势：
- 前几个 epoch IC 明显低于 baseline → kill
- IC 长期震荡无上升趋势 → kill
- 过拟合（train loss↓↓ 但 IC 不升反降）→ kill

## 4. 数据划分防泄露

```
全部日期按时间排序（绝对不能随机划分！）
├── 前 70%      → 训练集
├── 70% - 85%   → 早停/评估集（IC 早停 + best checkpoint + 回测）
└── 85% - 100%  → Hold-out 集（预留样本外验证，不参与任何调参）
```

### 4.1 时序交叉验证

```python
# 扩展窗口法（Expanding Window）
for i in range(n_splits):
    train = data[:start + i * step]
    test = data[start + i * step: start + (i+1) * step]

# 滚动窗口法（Rolling Window）
for i in range(n_splits):
    train = data[i * step: start + i * step]
    test = data[start + i * step: start + (i+1) * step]
```

**A 股推荐**：扩展窗口 + 验证/测试集间留 gap（如 5 个交易日），防止短期自相关伪信号。

## 5. 数据质量工程

### 5.1 数据泄露检查清单

| 泄露类型 | 描述 | 检查方法 |
|----------|------|----------|
| **前视偏差** | 使用未来信息（财报发布前用财报数据） | 检查时间戳 vs 实际可获取时间 |
| **存活偏差** | 只用幸存股票，忽略退市股票 | 确认数据集含退市股票 |
| **选择偏差** | 训练/测试有时间重叠 | 严格按时间划分 |
| **标签泄露** | 特征计算包含标签信息 | 检查特征计算代码 |
| **归一化泄露** | 用全量数据做归一化 | 归一化只用训练集统计量 |

### 5.2 A 股数据特殊问题

| 问题 | 说明 | 处理 |
|------|------|------|
| **涨跌停** | 涨跌停日因子值失真 | 涨跌停日剔除或特殊标记 |
| **停牌** | 停牌期间无有效交易数据 | 停牌日因子值 NaN，复牌后留缓冲期 |
| **ST / PT** | 交易规则不同（5% 涨跌幅） | 从股票池中剔除 |
| **次新股** | IPO 效应（上市初期异常波动） | 上市天数 <120 天剔除 |
| **财报发布延迟** | 年报最迟 4/30，一季报最迟 4/30 | 必须用 point-in-time 数据 |
| **分红除权** | 除权日价格跳变 | 使用后复权价格计算收益 |
| **指数成分变更** | 成分股每半年调整 | 用当时的成分股列表，非当前 |
