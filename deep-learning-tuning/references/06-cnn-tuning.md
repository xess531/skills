# CNN 专项调参指南

> 来源：阿里云 CNN 超参数优化、目标检测 Bag of Freebies、多篇 CV 调参博客

## 一、CNN 架构超参数速查

| 参数 | 推荐范围/默认 | 经验法则 |
|------|---------------|----------|
| **卷积核大小** | 3×3（主流）；1×1（通道变换）；5×5/7×7（首层大感受野） | 3×3 叠加优于单个大卷积核（VGG 发现） |
| **通道数** | 逐层递增：64→128→256→512 | 每次下采样（stride=2 或 pool）时通道数×2 |
| **网络深度** | ResNet-18/34/50/101 | 更深≠更好，需残差连接支撑；观察验证集性能饱和点 |
| **Pooling** | MaxPool 2×2（分类）；AvgPool（检测/分割的 neck） | Global Average Pooling 替代 FC 减少参数 |
| **Stride** | 首层可用 stride=2 快速下采样 | stride=2 下采样 vs MaxPool：前者减计算，后者保信息 |
| **Padding** | same padding（保持空间尺寸） | 输出尺寸 = (N + 2P - K) / S + 1 |

## 二、CNN 特有训练技巧

**Bag of Tricks（He et al. CVPR 2019）—— ResNet 从 75.3% → 79.29%：**

| Trick | 效果 | 说明 |
|-------|------|------|
| **大 mini-batch + 线性缩放 LR** | +0.3% | BS 从 256→1024，LR 同比×4 |
| **LR warmup** | 稳定训练 | 前 5 epoch 从 0 线性增到目标 LR |
| **零化 BN γ** | +0.2% | 残差分支末尾 BN 的 γ 初始化为 0 |
| **无 bias decay** | +0.2% | 只对 weight 做 WD，bias 和 BN 参数不做 |
| **余弦退火** | +0.4% | 替代 step decay |
| **Label Smoothing ε=0.1** | +0.3% | 防止过度自信 |
| **Mixup α=0.2** | +0.5% | 数据增强 |
| **知识蒸馏** | +1.0% | 大模型当老师 |

**CNN Batch Size 特殊注意：**
- CNN 对 batch size 敏感，**小 batch（16~128）通常泛化更好**
- 大 batch 可加速但需搭配 warmup + 线性缩放 LR + 无 bias decay
- BN 要求 batch 不能太小（<8 时统计量不稳定，换 Group Norm）

## 三、CNN 可视化诊断

| 检查项 | 正常表现 | 异常信号 |
|--------|----------|----------|
| 卷积核权重 | 平滑、有结构的滤波器 | 大量噪声纹理 → 训练不充分或正则化太弱 |
| 特征图激活 | 稀疏、局部激活 | 大片全零 → Dying ReLU 或 LR 太大 |
| 不同输入的激活差异 | 不同输入有不同激活模式 | 所有输入激活相似 → 模型没学到有用特征 |

## 四、目标检测/CV 专项

| 技巧 | 内容 | 提升幅度 |
|------|------|----------|
| **视觉连贯 MixUp** | 两张图直接叠加（取最大宽高，不 resize），解决特征干扰 | 提升召回率 |
| **多尺度训练** | 从 {320, 352, ..., 608} 随机选尺度训练 | 增强尺度鲁棒性 |
| **跨卡同步 BN** | 多 GPU 时全局统计 BN，改善小 BS 效果 | +0.5~1% |
| **Label Smoothing** | 分类头使用 y'=y(1-α)+α/K | 提升泛化 |
