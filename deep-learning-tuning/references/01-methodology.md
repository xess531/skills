# 科学调参方法论

## 一、增量调优策略（核心循环）

```
确定目标 → 设计实验 → 运行 → 分析结果 → 决策（采纳/放弃）→ 重复
```

**关键原则：**
- 大部分时间花在**探索**（理解问题结构、超参敏感性），而非**利用**（只盯最低验证误差）
- 理解问题有助于识别无关特征、避免不必要的复杂性
- 每次只改变一个变量，确保可以准确归因

## 二、超参数三分类

在设计实验时，必须将超参数分为三类：

| 类别 | 定义 | 举例 |
|------|------|------|
| **科学超参数** | 本轮实验要测量的目标变量 | 隐藏层数、模型宽度 |
| **干扰超参数** | 为公平比较必须同步优化的参数 | 学习率（最佳 LR 通常随模型大小变化） |
| **固定超参数** | 本轮保持不变的参数 | 批量大小、优化器类型 |

**规则：**
- 优化器超参数通常是**干扰参数**，很少作为科学参数
- 正则化强度是**干扰参数**，但"是否使用某正则化技术"是**科学参数**
- 架构参数（层数等）通常是**科学参数**或**固定参数**

## 三、搜索策略

- **探索阶段优先使用准随机搜索**（quasi-random search），优于贝叶斯优化
  - 原因：非自适应采样便于事后分析，结果可复现，并行效率高
- 如果最佳点落在搜索空间**边界**，说明空间不够大，需要扩展
- 使用**对数尺度**搜索学习率等参数（如 1e-5 到 1e-1）

## 四、选择模型架构

- 尽可能复用**已有的、经过验证的标准架构**（ResNet, Transformer, BERT 等）
- 不要一开始就构建自定义模型
- 找到与当前问题最接近的论文并**复现其模型**作为起点
- 确保预处理流程（归一化、Tokenization）与基准一致

## 五、调参决策框架

### 何时采纳新配置

- 改进效果必须**显著大于**重训练方差（random seed variance）
- 必须考虑**研究方差**（实验设计本身的偶然性）
- 只有当改进效果大于引入的复杂度时才更新基准

### 调参优先级总排序

```
1. 模型架构（决定性能上限）
2. 学习率（最重要的单个超参数）
3. 训练数据量和质量
4. 批量大小（影响训练效率）
5. 训练步数 / epochs
6. 学习率衰减策略
7. 正则化参数（dropout, weight decay）
8. 优化器高级参数（β₁, β₂, ε）
9. 其他（label smoothing, 数据增强强度等）
```

### 快速决策表

| 问题 | 首先尝试 | 然后尝试 |
|------|----------|----------|
| 模型不收敛 | 降低 LR / 检查代码 | 增加 warmup / 梯度裁剪 |
| 过拟合 | 增加 dropout / weight decay | 增加数据 / 数据增强 |
| 欠拟合 | 增大模型 / 降低正则化 | 提高 LR / 训练更久 |
| 训练太慢 | 增大 batch size | 使用更快的优化器 |
| 结果不稳定 | 多 seed 验证 | 增加 warmup |
| IC/指标提升不明显 | 检查特征工程 / 数据质量 | 尝试不同架构 |

## 六、实验管理最佳实践

### 实验记录

每次实验必须记录：
- 超参数配置（完整 config）
- 训练/验证 loss 曲线
- 核心指标（IC, Sharpe, Top-K 收益率等）
- 训练时长
- 是否出现异常（spike, NaN 等）

### 对比方法

- 使用**隔离图**（isolation plot）：固定干扰参数最优值后，只看科学参数的效果
- 多 seed 取平均，报告均值 ± 标准差
- 用表格对比前后变化，标注关键差异

### 检查点策略

- 按固定步数间隔评估，而非时间间隔
- 评估时 batch size ≥ 训练 batch size
- 训练结束后选**历史最佳 checkpoint**，而非最后一个
- BatchNorm 需注意多主机训练时的统计同步问题

## 七、给调参者的忠告

1. **耐心**：调参是迭代过程，不要期望一次到位
2. **记录一切**：没有记录的实验等于没做
3. **先跑通再优化**：确保 pipeline 正确比调参更重要
4. **信任数据而非直觉**：直觉可以指导搜索方向，但决策要看数据
5. **知道何时停止**：如果多轮调参收益递减，说明需要从别的维度突破（数据/特征/架构）
6. **简单优先**：复杂的方案如果没有显著优于简单方案，就用简单的
7. **多 seed 验证**：单次实验有偶然性，至少 3 个 seed 取平均
8. **先探索后利用**：大部分时间花在理解参数敏感性上，最后才冲刺最优值
9. **关注交互效应**：改了 LR 别忘了重新检查 WD 和 warmup
10. **压缩与部署是调参的延续**：剪枝/蒸馏后仍需微调，不要以为训练结束就可以了
