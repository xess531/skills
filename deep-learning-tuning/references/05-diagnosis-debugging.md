# 训练诊断与调试

## 一、训练失败模式百科

### Loss 异常

| 现象 | 原因 | 修复 |
|------|------|------|
| **Loss = NaN** | 学习率太大 / 初始化错误 / 数据含 NaN / FP16 溢出 | 降 LR → 检查数据 → 检查初始化 → 开启 GradScaler |
| **Loss = 常数不动** | LR 太小 / 梯度消失 / 模型 bug / 标签错误 | 检查梯度 → 提高 LR → 检查数据 pipeline |
| **Loss 震荡不收敛** | LR 太大 / batch 太小 / 数据噪声大 | 降 LR → 增大 batch → 增加 warmup |
| **Loss spike 后恢复** | 偶发梯度爆炸 / 异常样本 | 梯度裁剪 → 检查数据质量 |
| **Loss spike 后发散** | 严重不稳定 | 降 LR + 梯度裁剪 + warmup |
| **Loss 下降极慢** | LR 太保守 / 模型太小 / 优化器不适合 | 提高 LR → 增大模型 → 换优化器 |

### 梯度异常

| 现象 | 原因 | 修复 |
|------|------|------|
| **梯度消失** | 网络太深 + Sigmoid/Tanh / 初始化不当 | 用 ReLU 系列 + He 初始化 + 残差连接 + BN/LN |
| **梯度爆炸** | 深层网络 + 大 LR / 初始化方差过大 | 梯度裁剪 + 降 LR + 调整初始化 |
| **Dying ReLU** | 大量神经元输出恒为 0 | 换 Leaky ReLU / PReLU / GELU → 减小 LR |
| **梯度 norm 持续增长** | 不稳定的训练动态 | 梯度裁剪 + 检查架构 |

### 过拟合/欠拟合

| 现象 | 诊断 | 修复 |
|------|------|------|
| Train loss ↓↓ Val loss ↑ | 经典过拟合 | 增加正则化 / 数据增强 / 减小模型 |
| Train loss ↓ Val loss 平 | 轻度过拟合 | 增加 dropout / WD / early stopping |
| Train + Val loss 都高 | 欠拟合或过正则化 | 增大模型 / 降低正则化 / 检查数据 |
| Train loss 不动 | 严重欠拟合或 bug | 先验证模型能过拟合 10 个样本 |

## 二、训练不稳定性诊断与修复

### 不稳定性表现

- Loss 突然飙升（spike）然后可能恢复或直接发散
- 训练前期 loss 剧烈震荡
- 在较高学习率下无法收敛

### 修复手段（按优先级）

```
1. 学习率预热（Warmup）
   → 适用于早期不稳定
   → 从 0 线性增加到目标 LR

2. 梯度裁剪（Gradient Clipping）
   → 适用于梯度爆炸
   → 阈值设为梯度 norm 的 90% 分位数附近

3. 架构调整
   → 确保 Normalization 位于残差连接内部：x + f(Norm(x))
   → 而非外部：Norm(x + f(x))

4. 降低学习率
   → 最后手段，会损失训练效率
```

## 三、训练曲线诊断

### 看什么

| 曲线特征 | 诊断 | 行动 |
|----------|------|------|
| 训练 loss 不下降 | 学习率太低 / 模型容量不足 / Bug | 检查代码 → 提高 LR → 增大模型 |
| 训练 loss 下降但验证 loss 不降 | 过拟合 | 增加正则化 / 增加数据 |
| 训练 loss 和验证 loss 都很高 | 欠拟合或过正则化 | 降低正则化 / 增大模型 / 检查数据 |
| 验证 loss 先降后升 | 过拟合 | 用 early stopping / 增加正则化 |
| Loss 出现 spike | 训练不稳定 | 见不稳定性修复方案 |
| 训练 loss 极低但验证 loss 非常高 | 严重过拟合 | 强正则化 + 检查数据泄露 |

### 关键调试步骤

**过拟合一小批数据**：如果模型无法在极少量样本（如 10 个）上达到接近 0 的 loss，说明代码实现或模型结构有 Bug。这是最重要的 sanity check。

## 四、偏差-方差诊断框架

> 来源：博客园深度学习模型调优方法

### 四象限诊断表

| 偏差 | 方差 | 状态 | 优化方向 |
|------|------|------|----------|
| **高** | **低** | 欠拟合 | 增大模型容量 / 更复杂架构 / 训练更久 / 降低正则化 |
| **低** | **高** | 过拟合 | 增加数据 / 数据增强 / 正则化(Dropout/WD) / Early Stopping |
| **低** | **低** | 理想状态 | 保持 |
| **高** | **高** | 模型不匹配 | 换模型架构 / 检查数据质量 / 重新审视问题建模 |

### 三阶段调优逻辑

```
第一步：初步分析 → 选择合适的损失函数和优化器
第二步：训练集优化 → 确保模型能拟合训练集（调 LR、激活函数、模型大小）
第三步：验证集优化 → 提升泛化（正则化、Dropout、Early Stopping、数据增强）
```

**黄金法则：** 如果训练集效果都不好，不要急着加正则化——先让模型有能力拟合数据，再控制过拟合。
