# 优化器、学习率与批量大小

## 一、选择优化器

### 推荐

| 优化器 | 特点 | 适用场景 |
|--------|------|----------|
| **Adam / NAdam** | 通用性强，对超参较鲁棒 | 默认首选 |
| **AdamW** | Adam + 解耦权重衰减 | 需要正则化时 |
| **SGD + Nesterov Momentum** | 可能更好的泛化性 | Adam 调优遇瓶颈后尝试 |

### Adam 超参数调优优先级

根据实验预算分配：

| 预算 | 调什么 |
|------|--------|
| **< 10 次试验** | 仅调 **learning rate** |
| **10-25 次** | 调 **learning rate** + **β₁** |
| **25+ 次** | 调 **learning rate** + **β₁** + **ε** |
| **远超 25 次** | 再考虑调 **β₂** |

### 初始建议值

- Adam 默认：`lr=1e-3, β₁=0.9, β₂=0.999, ε=1e-8`
- 项目初期建议固定 `β₁, β₂, ε`，仅调学习率
- 后期精调时再逐步解锁其他参数

## 二、选择批量大小

### 核心原则

**批量大小主要影响训练速度，不应作为直接调优验证集性能的超参数。**

### 选择策略

- 在硬件允许范围内选择**尽可能大的**批量大小
- 较大批量可加速训练（利用并行计算），但可能损害泛化
- **批量大小改变 → 必须重新调整学习率和正则化参数**
- 梯度累积模拟更大批量，但**不提供吞吐量优势**，通常不建议

### 批量大小与学习率的关系

- 经验法则：批量大小 ×2 → 学习率也需适当增大（线性缩放规则）
- 但线性缩放并非总是成立，大批量需要额外验证

## 三、学习率与衰减策略

### 学习率衰减

| 策略 | 适用场景 |
|------|----------|
| **余弦衰减 (Cosine Decay)** | 默认推荐，适用于大部分场景 |
| **线性衰减 (Linear Decay)** | 另一个不错的默认选择 |
| **恒定学习率** | 调参初期快速探索时使用 |

- 避免使用基于验证误差的"手动干预式"衰减，除非过程完全自动化
- 学习率衰减通常在训练**末尾阶段**执行效果最好

### 学习率预热 (Warmup)

- 适用于**早期训练不稳定**的情况
- 将学习率从 0（或很小值）线性/指数增加到目标值
- 典型预热步数：总步数的 5%~10%，或几百到几千步
- 如果预热后仍不稳定，考虑降低目标学习率

---

## 四、选择训练步数

### 两种场景

| 场景 | 策略 |
|------|------|
| **计算受限** | 训练越久越好；可以先短跑找参数，再长跑定结果 |
| **计算充足** | 找到性能饱和点，避免浪费 |

### 确定步数的方法

1. 先用**无正则化 + 恒定学习率**跑实验，找到能完美拟合训练集的步数 N
2. 以 N 为基准，引入正则化/数据增强后适当增加步数
3. 使用**回顾性最优检查点选择**：训练结束后选历史最佳 checkpoint，而非最后一个
