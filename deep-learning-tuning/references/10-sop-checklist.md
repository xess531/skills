# 调参 SOP（标准操作流程）

## 一、完整调参 Checklist

```
□ 第零步：数据检查
  □ 数据中有无 NaN / Inf / 异常值？
  □ 标签分布是否合理（类别不平衡？）
  □ 训练/验证/测试集划分是否正确（时间序列不能随机 split）？
  □ 预处理是否与参考实现一致（归一化均值/方差、tokenization）？

□ 第一步：Sanity Check
  □ 模型能在 10 个样本上 loss → 0？（验证代码无 bug）
  □ 恒定学习率下训练 loss 是否在下降？
  □ 梯度 norm 是否在合理范围？（不为 0，不爆炸）

□ 第二步：建立 Baseline
  □ 使用默认/论文推荐的超参数
  □ 记录：train loss, val loss, 核心指标, 训练时长
  □ 这就是后续所有实验的对照组

□ 第三步：确定调参优先级
  □ 学习率（最先调）
  □ 批量大小（确定后基本固定）
  □ 学习率衰减 + warmup
  □ 正则化参数（dropout, weight decay）
  □ 模型架构参数（层数、宽度）

□ 第四步：系统化搜索
  □ 每次只改一个（或一组强关联的）变量
  □ 用准随机搜索 / 贝叶斯优化
  □ 多 seed 验证（≥3 个 seed），报告均值±标准差
  □ 最佳点在搜索边界？→ 扩大范围重搜

□ 第五步：验证与决策
  □ 改进是否显著大于 seed 方差？
  □ 改进是否值得引入的额外复杂度？
  □ 更新 baseline，进入下一轮

□ 第六步：最终确认
  □ 在最佳配置上跑 3~5 个 seed
  □ 选最佳 checkpoint（非最后一个）
  □ 完整记录最终配置和结果
```

## 二、快速排错流程

```
Loss = NaN？
  → 降 LR × 10 → 检查数据有无 NaN → 检查初始化 → 开 GradScaler (FP16)

Loss 不动？
  → 检查梯度（是否全 0）→ 提高 LR → 检查 data pipeline → 验证 10 样本过拟合

Loss 震荡？
  → 降 LR → 增大 batch → 增加 warmup → 梯度裁剪

过拟合（train↓ val↑）？
  → 增加 dropout → 增加 WD → 数据增强 → 减小模型

欠拟合（train+val 都高）？
  → 增大模型 → 降低正则化 → 提高 LR → 训练更久

训练极慢？
  → 增大 batch → 检查 data loader（num_workers, prefetch）→ 混合精度
```
