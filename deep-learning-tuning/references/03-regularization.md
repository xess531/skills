# 正则化、初始化与超参数交互效应

## 一、正则化策略

### 常用正则化手段

| 技术 | 典型范围 | 注意事项 |
|------|----------|----------|
| **Dropout** | 0.05 ~ 0.5 | 过高会欠拟合；与 BatchNorm 搭配需小心 |
| **Weight Decay** | 1e-6 ~ 1e-2 | AdamW 中解耦权重衰减更优 |
| **数据增强** | 视任务而定 | "是否使用"是科学参数，"强度"是干扰参数 |
| **Label Smoothing** | 0.05 ~ 0.2 | 分类任务常用 |
| **Early Stopping** | 按验证集性能 | 保守但有效 |

### 过正则化的信号

- 训练 loss 远高于预期（模型连训练集都拟合不好）
- 验证 loss 和训练 loss 同步下降但都很高
- 降低正则化强度后训练 loss 显著下降

### 欠正则化的信号

- 训练 loss 很低但验证 loss 偏高（明显的 train-val gap）
- 训练后期验证 loss 开始上升
- 增加正则化后验证 loss 下降

## 二、1Cycle Policy 与 LR Range Test（Leslie Smith 方法）

> 来源：Leslie N. Smith, "A Disciplined Approach to Neural Network Hyper-parameters" (arXiv:1803.09820)

### LR Range Test（学习率范围测试）

**用途：快速找到合适的学习率范围，替代昂贵的网格搜索。**

**操作步骤：**
1. 从一个极小的学习率（如 1e-7）开始训练
2. 每个 mini-batch 后线性（或指数）增大学习率
3. 记录 loss 随 LR 的变化曲线
4. 运行几百到几千步即可

**如何解读：**
- Loss 先下降，然后在某个点开始急剧上升（发散）
- **最大 LR**：取 loss 开始上升前的那个值（最低 loss 对应的 LR）
- **最小 LR**：通常取最大值的 1/3 到 1/10

```
Loss
  |    ____
  |   /    \
  |  /      \    ← 发散点
  | /        \
  |/          \___
  +---------------→ Learning Rate
       ↑        ↑
    最小LR    最大LR
```

### 1Cycle Policy（1周期策略）

**核心思想：用一个周期完成训练，结合"课程学习"和"模拟退火"，实现超收敛。**

**操作步骤：**
1. **升温阶段**：LR 从最小值线性增加到最大值（约占总步数 45%）
2. **降温阶段**：LR 从最大值线性减小到比最小值更低几个数量级（约占总步数 45%）
3. **退火阶段**：LR 降至极小值完成最后的精调（约占总步数 10%）

**效果：**
- 训练迭代次数可从 80K 减少到 10K 并达到同等或更好性能
- 大 LR 本身就是一种正则化，减少过拟合

### Cyclical Momentum（周期性动量）

**核心规则：LR 增加时，Momentum 应减小；LR 减小时，Momentum 增加。**

| LR 阶段 | Momentum |
|----------|----------|
| LR 从小增大 | Momentum 从 0.95 减到 0.85 |
| LR 从大减小 | Momentum 从 0.85 增到 0.95 |
| 固定 LR | 常数 Momentum 0.9~0.99 |

### 正则化总量平衡法则

**核心：所有正则化的总效果必须平衡。增加某种正则化时，必须减少其他正则化。**

- 使用大 LR（它本身是正则化）→ 减少 Weight Decay
- 使用 1Cycle 大 LR → 最优 WD 通常**比固定小 LR 时更小**
- 增加 Dropout → 可以减少 WD
- 增加数据增强 → 可以减少 Dropout

## 三、权重初始化

### 选择指南

| 激活函数 | 推荐初始化 | 公式 |
|----------|------------|------|
| **Sigmoid / Tanh** | Xavier (Glorot) | `W ~ U[-v, v], v = sqrt(6 / (fan_in + fan_out))` |
| **ReLU / Leaky ReLU** | He (Kaiming) | `W ~ N(0, sqrt(2 / fan_in))` |
| **SELU** | LeCun | `W ~ N(0, sqrt(1 / fan_in))` |
| **通用/不确定** | He 初始化 | 最安全的默认选择 |

### 实用规则

- 偏置（bias）通常初始化为 **0**
- LSTM 遗忘门偏置初始化为 **1**（让信息在训练初期能流通）
- 使用 BatchNorm 时，对初始化的敏感度会降低
- 如果训练初期 loss 不下降或 NaN，**首先检查初始化**
- Transformer 中通常使用截断正态分布 + 缩放因子

### 错误初始化的症状

| 症状 | 可能的原因 |
|------|-----------|
| Loss 从第一步就是 NaN | 初始化方差太大 → 梯度爆炸 |
| Loss 不动（平坦） | 初始化方差太小 → 梯度消失 |
| 深层网络后几层输出全 0 | 使用 ReLU + Xavier → 换 He 初始化 |
| 训练极慢 | 初始化不匹配激活函数 |

## 四、超参数交互效应

**调参最大的陷阱不是单个参数，而是参数之间的联动关系。**

### 交互矩阵

| 参数 A ↓ \ 参数 B → | Learning Rate | Batch Size | Weight Decay | Dropout | Warmup |
|----------------------|---------------|------------|--------------|---------|--------|
| **Learning Rate** | — | ⚠️ 线性缩放规则 | ⚠️ 大LR需减WD | 弱交互 | ⚠️ 高LR需长warmup |
| **Batch Size** | ⚠️ BS×2→LR需增 | — | ⚠️ 大BS可能需更多正则 | 弱交互 | 弱交互 |
| **Weight Decay** | ⚠️ 大LR需减WD | ⚠️ 需协同调整 | — | ⚠️ 叠加过正则化 | 弱交互 |
| **Dropout** | 弱交互 | 弱交互 | ⚠️ 叠加过正则化 | — | 弱交互 |
| **Model Size** | ⚠️ 大模型可能需不同LR | ⚠️ 大模型需更多显存 | ⚠️ 大模型需更强正则 | ⚠️ 大模型需更多dropout | 弱交互 |

### 关键交互规则

**LR × Batch Size：**
- 经典线性缩放规则：BS ×2 → LR ×2
- **但大 BS（>4096）时线性缩放常失效**，需要 warmup + 更保守的缩放
- 小 BS（<64）对 LR 更敏感，需要更精细的 LR 搜索

**Weight Decay × Dropout：**
- 两者都是正则化，**叠加可能导致过正则化**
- 如果同时使用，建议先固定一个调另一个
- 实践中常见的好组合：中等 WD + 低 Dropout，或 低 WD + 中等 Dropout

**LR × Weight Decay（AdamW 特殊性）：**
- Adam 中 WD 和 LR 耦合：实际正则化强度 = WD × LR
- **AdamW 解耦了两者**，所以用 AdamW 时 WD 的效果更可预测
- 用 Adam 调 WD 时，改变 LR 后必须重新调 WD

**Model Size × 正则化：**
- 大模型容量大 → 更容易过拟合 → 需要更强正则化
- 小模型容量小 → 正则化过强会欠拟合 → 适当减弱正则化
