# 超参数自动搜索方法论

> 来源：火山引擎超参数自动化调优详解 + Google Tuning Playbook

## 一、四种搜索方法对比

| 方法 | 原理 | 优点 | 缺点 | 推荐场景 |
|------|------|------|------|----------|
| **网格搜索** | 穷举所有组合 | 不漏组合 | 维度灾难，指数爆炸 | ≤3 个参数 + 离散值 |
| **随机搜索** | 参数空间随机采样 | 比网格高效，不受维度限制 | 可能错过最优区域 | 初步探索阶段 |
| **准随机搜索** | 低差异序列采样（Sobol/Halton） | 比随机覆盖更均匀，可复现 | 实现略复杂 | **Google Playbook 推荐** |
| **贝叶斯优化（TPE/GP）** | 用历史结果建代理模型指导下次采样 | 最高效，收敛快 | 串行依赖，不利并行；初始采样不够时表现差 | 精调阶段（>20次实验后） |

---

## 二、两阶段搜索策略（推荐）

```
阶段一（探索）：准随机搜索
  - 大范围搜索（对数尺度）
  - 目标：理解参数敏感性，确定有效范围
  - 如果最优点在边界 → 扩大搜索空间

阶段二（利用）：贝叶斯优化或精细准随机
  - 缩小范围到阶段一发现的有效区域
  - 目标：找到精确最优组合
  - 工具推荐：Optuna / Hyperopt / Ray Tune
```

## 三、实用工具

| 工具 | 语言 | 算法 | 特点 |
|------|------|------|------|
| **Optuna** | Python | TPE + 多种 | API 简洁，支持剪枝（Pruning），推荐首选 |
| **Hyperopt** | Python | TPE | 经典库，搜索空间定义灵活 |
| **Ray Tune** | Python | 多种 + 调度 | 分布式调参，适合大规模 |
| **Weights & Biases Sweeps** | Python | 多种 | 实验追踪一体化 |
| **Keras Tuner** | Python | 多种 | Keras 生态内使用最方便 |

## 四、搜索空间定义技巧

- **学习率/Weight Decay**：用对数均匀分布 `hp.loguniform`
- **Dropout/Label Smoothing**：用线性均匀分布 `hp.uniform`
- **层数/头数/通道数**：用整数选择 `hp.choice` 或 `hp.quniform`
- **优化器类型/激活函数**：用分类选择 `hp.choice`
- **条件参数**：Adam 和 SGD 各有独立参数空间，用嵌套定义
