# Deep Research — 横纵分析法深度研究 Skill

一个让 AI Agent 自动产出万字级深度研究报告的 Skill。

给它一个名字——产品、公司、技术概念、人物——它会联网搜索、沿时间轴还原完整发展脉络、在当下截面与竞品横向对比，最后交叉两条轴给出独到洞察，输出一份结构清晰的 Markdown 报告。

## 方法论

横纵分析法借鉴了数字生命卡兹克（Khazix）提出的研究框架，融合了几个学科的核心思想：

- **索绪尔的历时-共时分析**：语言学家索绪尔区分了「历时」（沿时间演变）和「共时」（同一时间截面的结构），这成为纵横两条轴的理论根基
- **社会科学的纵向-横截面研究设计**：纵向追踪同一对象随时间的变化，横截面在同一时间点比较不同对象
- **商学院案例研究法与竞争战略分析**：波特五力、竞品矩阵等工具的实战应用

核心原则很简单：**纵轴追时间深度，横轴追同期广度，交汇出判断。**

## 特性

**🔍 信息源自适应**

不绑定任何特定工具。每次执行时自动探测当前 Agent 环境中可用的搜索能力——MCP 工具、已安装的 Skill/插件、内置搜索函数、知识库——然后用能力最强的组合去获取信息。

**📏 研究深度自适应**

根据研究对象的信息密度自动判断报告深度：

| 等级 | 适用对象 | 总篇幅 |
|------|---------|--------|
| 重量级 | 腾讯、React、Transformer 这类信息极丰富的对象 | 2-3万字 |
| 标准级 | Cursor、Mamba 这类信息充分的对象 | 1-1.8万字 |
| 轻量级 | 刚发布的新工具、冷门概念 | 0.6-1万字 |

**🔀 平台无关**

不依赖任何特定 Agent 框架。WorkBuddy、Claude Code、Cursor、Windsurf、自建 Agent——只要能加载 Skill 就能跑。并行搜索是推荐而非必须，不支持并行的环境会自动退化为顺序执行。

**✍️ 不是冷冰冰的报告**

写作风格追求叙事驱动而非罗列驱动。纵向部分有故事弧线，不是年表流水账；横向部分讲的是每个竞品「活成了什么样」，不是参数对照表的文字版。有明确的 AI 味禁区——「综上所述」「赋能」「在当今 AI 快速发展的时代」这些都不会出现。

## 工作流

```
研究对象 → 判断深度等级 → 能力探测 → 信息收集（纵向+横向+补充）
  → 纵向分析（完整发展脉络）→ 横向分析（竞品对比）
  → 横纵交汇洞察 → 保存 Markdown 报告
```

## 使用

把 `SKILL.md` 放到你的 Agent 的 Skill 目录下即可。触发方式：

- "帮我研究一下 Cursor"
- "深度研究 Transformer 的前世今生"
- "帮我做个竞品分析，看看 XX 和 YY"
- "这个东西是怎么回事？"（上下文暗示需要深度研究时自动触发）
