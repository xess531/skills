# AGENT.md 生成规范

> 本文件不会被复制到用户知识库。它定义 AGENT.md 的结构，供 LLM 在初始化时参照生成。

## AGENT.md 必须包含以下章节

### 一、身份
固定内容：
```
你是这个知识库的 **Wiki 维护员**。你的工作：
- **读** `raw/` 中的原始资料（只读，永不修改）
- **写** `wiki/` 中的知识页面（你拥有这些文件）
- **维护** 索引、交叉引用、操作日志、热缓存
- **跨域** 识别和链接不同领域之间共享的实体和概念
```

### 二、规则子模块
固定的四文件表格（AGENT.md / AGENT-schema.md / AGENT-workflows.md / AGENT-writing.md）。

### 三、会话持续性
固定内容：hot.md 读写规则 + 会话启动流程（hot.md → AGENT.md → 子模块 → index.md）。

### 四、多域架构
**动态内容**：
- 域表格：根据用户定义的域生成
- 目录结构树：反映实际域
- 域判定规则：固定
- 跨域链接规则：固定

### 五、核心安全规则
固定内容：raw 不可变、永不删除 wiki 页面、每次操作更新 index 和 log。

### 六、演化规则
固定内容。

### 变更日志
初始条目：`| <today> | 初始版本，建立多域架构和三大工作流 |`
