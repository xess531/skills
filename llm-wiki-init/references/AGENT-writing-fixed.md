# AGENT-writing.md 固定部分（写作规范）

> 生成 AGENT-writing.md 时，先复制本文件的写作规范部分，然后根据域信息动态追加标签体系。

## 写作规范

1. **语言**：中文为主，专业术语保留英文原文
2. **风格**：精确、简洁、信息密度高。不要空泛的描述，要具体的事实和数据
3. **引用**：所有论述必须可追溯到 `raw/` 中的来源。使用 `(来源: [[wiki/<domain>/sources/xxx]])` 格式
4. **交叉链接**：使用 Obsidian 的 `[[wikilink]]` 格式。跨域链接写全路径如 `[[wiki/<domain>/concepts/xxx]]`。首次出现时链接，后续不重复
5. **矛盾处理**：发现矛盾时不要删除旧内容，用 blockquote 标注：
   ```
   > ⚠️ **矛盾**：来源 A 认为 X，但来源 B 认为 Y。待进一步验证。
   ```
6. **文件命名**：全部小写，用连字符分隔，如 `react-hooks.md`、`elon-musk.md`
7. **标签约定**：使用小写英文，域级标签作为一级分类

## 标签体系（动态生成部分）

> 以下内容由 LLM 在初始化时根据用户定义的域自动生成。

### 生成规则

1. 先列域级标签（每个域 ID 作为一级标签）
2. 对每个域，根据其描述生成 6-10 个二级标签
3. 标签格式：小写英文、连字符分隔、每个标签带 `# 中文注释`
4. 标签应覆盖该域最常见的子主题分类
